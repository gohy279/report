\section{Цель работы. Постановка задачи}

\textbf{Цель работы:} создание веб-приложения «Team Finder» --- платформы, на которой разработчики, дизайнеры и другие специалисты смогут находить единомышленников для совместной работы над pet-проектами. Зарегистрированные пользователи получат возможность размещать идеи проектов, собирать команду и откликаться на опубликованные предложения.

\textbf{Постановка задачи:} необходимо реализовать следующие функциональные возможности:

\begin{enumerate}
    \item Работа с пользователями. Создать кастомную модель пользователя с аутентификацией по email. Реализовать страницы регистрации, входа, выхода, просмотра и редактирования профиля, смены пароля.
    \item Пагинация. Подключить к проекту пагинацию и настроить вывод не более 12 проектов на главную страницу и на страницу списка пользователей.
    \item Аватарки пользователей. Добавить возможность загрузки аватарки. При регистрации должна автоматически генерироваться первоначальная аватарка с первой буквой имени.
    \item Создание проектов. У зарегистрированных пользователей должна быть возможность создавать новые проекты.
    \item Редактирование проектов. Автор проекта должен иметь возможность изменить его параметры.
    \item Система участия в проектах. Реализовать возможность присоединяться к проектам и отказываться от участия.
    \item Управление статусом и навыками проекта. Автор может завершить проект. Владелец проекта может добавлять и удалять навыки, необходимые проекту.
    \item Фильтрация проектов по навыкам. На странице списка проектов должна быть панель фильтрации по необходимым навыкам.
    \item Для выполнения задания необходимо использовать view-классы (CBV).
\end{enumerate}

\subsection{Пагинация}

В проекте настроена пагинация на главной странице проектов и на странице списка пользователей.
Списки делятся на страницы по 12 элементов с помощью штатного механизма \texttt{ListView} и атрибута \texttt{paginate\_by}.
Константа \texttt{DEFAULT\_PAGINATION\_LIMIT} вынесена в отдельный файл \texttt{constants.py}.

View-класс \texttt{ProjectListView} \texttt{projects/views.py}:
 
\begin{lstlisting}
class ProjectListView(ListView):
    model = Project
    template_name = 'projects/project_list.html'
    context_object_name = 'projects'
    paginate_by = constants.DEFAULT_PAGINATION_LIMIT

    def get_queryset(self):
        queryset = Project.objects.select_related('owner').prefetch_related(
            'participants',
            'skills'
        )
        skill_name = self.request.GET.get('skill')
        if skill_name:
            queryset = queryset.filter(skills__name=skill_name.strip())
        return queryset

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['all_skills'] = Skill.objects.all()
        skill_name = self.request.GET.get('skill')
        if skill_name:
            context['active_skill'] = Skill.objects.filter(
                name=skill_name.strip()).first()
        else:
            context['active_skill'] = None
        return context
\end{lstlisting}

class ProjectEditView(LoginRequiredMixin, UpdateView):
    model = Project
    form_class = ProjectCreateForm
    template_name = 'projects/create-project.html'
    pk_url_kwarg = 'project_id'

    def get_success_url(self):
        return reverse_lazy('projects:project_detail',
                            kwargs={'project_id': self.object.pk})

    def dispatch(self, request, *args, **kwargs):
        obj = self.get_object()
        if obj.owner != request.user:
            raise PermissionDenied(
                'Вы не можете редактировать чужой проект!')
        return super().dispatch(request, *args, **kwargs)

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['is_edit'] = True
        return context
\end{lstlisting}

\subsection{Участие в проектах}

В проекте реализована система участия в проектах.
Авторизованные пользователи могут присоединяться к чужим проектам или отказываться от участия.
Связь между пользователем и проектом реализована через ManyToMany-поле \texttt{participants} в модели \texttt{Project}.
Взаимодействие реализовано через POST-запросы с JSON-ответом.

Модель \texttt{Project} в \texttt{projects/models.py}:

\begin{lstlisting}
class Project(models.Model):
    class Status(models.TextChoices):
        OPEN = 'open', 'Открыт'
        CLOSED = 'closed', 'Закрыт'

    owner = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name='owned_projects',
        verbose_name='Автор проекта'
    )
    name = models.CharField(
        max_length=constants.PROJECT_NAME_MAX_LENGTH,
        verbose_name='Название проекта')
    description = models.TextField(
        blank=True, verbose_name='Описание проекта')
    participants = models.ManyToManyField(
        settings.AUTH_USER_MODEL,
        blank=True,
        related_name='participated_projects',
        verbose_name='Участники')
    github_url = models.URLField(
        blank=True, verbose_name='Ссылка на GitHub')
    created_at = models.DateTimeField(
        auto_now_add=True, blank=True, verbose_name='Дата создания')
    status = models.CharField(
        max_length=constants.PROJECT_STATUS_MAX_LENGTH,
        choices=Status.choices,
        default=Status.OPEN,
        verbose_name='Статус')
    skills = models.ManyToManyField(
        Skill,
        blank=True,
        related_name='projects',
        verbose_name='Навыки проекта')

    class Meta:
        verbose_name = 'Проект'
        verbose_name_plural = 'Проекты'
        ordering = ['-created_at']

    def __str__(self):
        return self.name
\end{lstlisting}

View-класс \texttt{AddParticipantView} в \texttt{projects/views.py}:

\begin{lstlisting}
class AddParticipantView(LoginRequiredMixin, View):
    def post(self, request, project_id):
        project = get_object_or_404(Project, pk=project_id)

        if project.status != Project.Status.OPEN:
            return JsonResponse({
                'status': 'error',
                'message': 'В завершенный проект нельзя вступить'
            }, status=HTTPStatus.BAD_REQUEST)

        if project.owner == request.user:
            return JsonResponse({
                'status': 'error',
                'message': 'Вы являетесь владельцем своего проекта'
            }, status=HTTPStatus.BAD_REQUEST)

        if project.participants.filter(pk=request.user.pk).exists():
            project.participants.remove(request.user)
            response_data = {
                'status': 'ok',
                'participant': None
            }
        else:
            project.participants.add(request.user)
            response_data = {
                'status': 'ok',
                'participant': {
                    'id': request.user.id,
                    'name': f'{request.user.first_name} {request.user.surname}',
                    'avatar': (
                        request.user.avatar.url
                        if request.user.avatar else None
                    )
                }
            }
        return JsonResponse(response_data)
\end{lstlisting}

\subsection{Закрытие проектов}

Реализование закрытие владельцем проекта, владелец может сменить его статус на <<closed>>.

View-класс \texttt{ProjectCompleteView} в \texttt{projects/views.py}:

\begin{lstlisting}
class ProjectCompleteView(LoginRequiredMixin, View):
    def post(self, request, project_id):
        project = get_object_or_404(Project, pk=project_id)

        if project.owner != request.user:
            return JsonResponse(
                {'status': 'error',
                 'message': 'Вы не можете завершить чужой проект!'},
                status=HTTPStatus.FORBIDDEN
            )

        if project.status != Project.Status.OPEN:
            return JsonResponse(
                {'status': 'error',
                 'message': 'Проект уже завершён'},
                status=HTTPStatus.CONFLICT
            )

        project.status = Project.Status.CLOSED
        project.save()
        return JsonResponse({'status': 'ok'})
\end{lstlisting}

View-классы в \texttt{skills/views.py}:

\begin{lstlisting}
class SkillAutocompleteView(View):
    def get(self, request):
        query = request.GET.get('q', '')
        skills = Skill.objects.filter(
            name__icontains=query
        )[:constants.SKILLS_PAGINATION_LIMIT]
        data = list(skills.values('id', 'name'))
        return JsonResponse(data, safe=False)


class AddSkillToProjectView(LoginRequiredMixin, View):
    def post(self, request, project_id):
        project = get_object_or_404(Project, id=project_id)
        if project.owner != request.user:
            return JsonResponse(
                {'error': 'У вас нет прав для редактирования этого проекта!'},
                status=HTTPStatus.FORBIDDEN
            )
        try:
            data = json.loads(request.body)
        except json.JSONDecodeError:
            return JsonResponse(
                {'error': 'Некорректный формат JSON'},
                status=HTTPStatus.BAD_REQUEST
            )
        skill_id = data.get('skill_id')
        name = data.get('name')
        if not skill_id and not name:
            return JsonResponse(
                {'error': 'Необходимо передать skill_id или name'},
                status=HTTPStatus.BAD_REQUEST
            )
        if skill_id:
            skill = get_object_or_404(Skill, id=skill_id)
        else:
            skill, _ = Skill.objects.get_or_create(name=name.strip())
        project.skills.add(skill)
        return JsonResponse({'id': skill.id, 'name': skill.name},
                            status=HTTPStatus.OK)


class RemoveSkillFromProjectView(LoginRequiredMixin, View):
    def post(self, request, project_id, skill_id):
        project = get_object_or_404(Project, pk=project_id)
        if project.owner != request.user:
            return JsonResponse(
                {'error': 'У вас нет прав для редактирования этого проекта!'},
                status=HTTPStatus.FORBIDDEN
            )
        skill = get_object_or_404(Skill, pk=skill_id)
        if project.skills.filter(id=skill.id).exists():
            project.skills.remove(skill)
            return JsonResponse({'status': 'ok'})
        return JsonResponse(
            {'status': 'error', 'message': 'Навыка нет в проекте'},
            status=HTTPStatus.BAD_REQUEST
        )
